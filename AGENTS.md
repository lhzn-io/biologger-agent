# Onboarding Portal & Gateway Architecture (biologger-agent)

> Inherits global standards, boundaries, and rules from the local agent config.

Welcome to the **biologger-agent** gateway orchestrator agent manual! This document is designed for coding agents to manage, verify, and maintain the ZeroClaw gateway stack.

---

## Agent Configuration Parameters

All runtime parameters are configured in the ZeroClaw configuration file:
*   **Path:** `config/.zeroclaw/config.toml`
*   **Default Gateway Port:** `42617`
*   **Target VLM Inference Host:** `http://host.docker.internal:8080/v1` (maps to host port 8080)
*   **Workspace Volume Path:** `/zeroclaw-data/workspace`

---

## SQLite Session Database Schema

To support multi-session conversations, history logs, and session deletion, ZeroClaw persists chats inside a local SQLite database:
*   **Database Path:** `config/.zeroclaw/sessions.db`
*   **Main Table:** `sessions`
*   **Schema Fields:**
    *   `id`: Primary key (Integer).
    *   `session_key`: Session identifier (Text). Sessions generated through the ZeroClaw gateway are stored with a `gw_` prefix (e.g. `gw_abc123`).
    *   `role`: The author role (Text, e.g. `user` or `assistant`).
    *   `content`: The raw text content of the message (Text).
    *   `created_at`: The creation timestamp (Text/DateTime).

### SQL Verification Examples

#### Check Saved Sessions
```sql
SELECT DISTINCT session_key FROM sessions WHERE session_key LIKE 'gw_%';
```

#### Fetch Session Transcript
```sql
SELECT role, content, created_at FROM sessions WHERE session_key = 'gw_session_id' ORDER BY id ASC;
```

#### Delete Session history
```sql
DELETE FROM sessions WHERE session_key = 'gw_session_id';
```

## Gateway Diagnostics & Verification Checklist

When onboarding on new hardware, follow these sequential steps to verify gateway health:

### 1. Probe Ports
Confirm the ZeroClaw container is listening on port `42617`:
```bash
curl -I http://localhost:42617/health
```

### 2. Verify VLM Integration
Confirm that the gateway can communicate with the backend VLM model server. Run a diagnostic request to the gateway completions route:
```bash
curl -X POST http://localhost:42617/webhook \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello"}'
```
Ensure the return payload includes a valid completion generated by the underlying model without timeouts.

### 3. Check Workspace Volumes
Confirm the dataset directories are correctly mounted and readable within the container sandbox:
```bash
docker compose exec biologger-agent ls -la /zeroclaw-data/datasets
```
Ensure files can be read in read-only mode (`ro`).
